# Пример нашего решения


В шаблоне есть два сервиса:
- `index` — получает сообщения и строит чанки для индексации;
- `search` — получает вопрос и возвращает `message_ids` найденных сообщений.

Это не эталонное решение по качеству поиска. Его задача — показать корректный контракт и базовый pipeline.

## Состав репозитория

- `index/main.py` — сервис индексации;
- `search/main.py` — сервис поиска;
- `data/Go Nova.json` — анонимизированный пример реального чата;
- `index/Makefile`, `search/Makefile` — локальная сборка и публикация образов.

## Что мы поменяли

Сделали два сервиса на FastAPI.

Первый — индекс-сервис.
Он режет сообщения на умные куски, не просто по 512 символов, а по смыслу, учитывая переписку и треды. Добавляет метаданные: кто написал, когда, кого упомянул.

Второй — поисковый сервис.
Ищет двумя способами одновременно:
— по смыслу (dense вектор)
— по ключевым словам (sparse вектор с лемматизацией)

Потом смешивает результаты, переранжирует моделью от NVIDIA и отдаёт.
При ошибках API — автоматически перезапрашивает.

Мы сделали 5 вещей, которых в базовом решении нет:

Чанкинг по смыслу, а не по кускам текста.

Разные векторы — одни для смысла, другие для точных слов.

Расширение запроса — через гипотетические ответы и ключевые слова.

Реренк топ-30 вместо топ-10.

Своя лемматизация — чтобы не тащить внешние библиотеки, и контейнеры меньше весили.

И всё это работало.

## Что мы планируем добавить

Мы планировали добавить четыре сильных улучшения, которые гарантированно подняли бы метрики.

Первое: индексация каждого сообщения отдельным чанком. Математически это оптимально для Recall — мы бы нашли ещё больше нужных сообщений.

Второе: HyDE multi-query — это когда по одному запросу нейросеть генерирует несколько гипотетических ответов, и мы ищем по всем сразу. Очень мощная техника.

Третье: полезное усиление — если в запросе точно совпадает имя или дата, мы искусственно поднимаем такие сообщения вверх.

Четвёртое: двухпроходной реранк — сначала дешёвая модель отсекает явно нерелевантное, потом дорогая аккуратно сортирует топ-10.
